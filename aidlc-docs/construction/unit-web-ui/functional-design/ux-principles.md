# UX 原則 — unit-web-ui

## プロダクト哲学（UI への落とし込み）

| 原則 | UI での表現 |
|------|-------------|
| ロボットは裁かない | 勝敗色、❌/✅、「悪い子」ラベル禁止 |
| 主役は人 | 子ども・先生の発話・判断が視覚的に主役；VRM は黒子（脇役） |
| 黒子 | アバターは聞く・整理する存在；派手な演出より落ち着いた idle |
| 先生が決める | `ai_disclaimer` を常時表示；自動処罰 UI なし |

## ペルソナ別

### 子ども

- **トーン**: 温かい、短い文、敬体すぎない（「〜だね」「〜してね」）
- **タップ領域**: 最小 44px
- **文字**: 16px 基準、行間 1.5
- **VRM**: 男性/女性を選べる（CharaTomo 同系）。選んだモデルは `localStorage` に保存
- **不安時**: エスカレーションは責めない — 「大丈夫？先生を呼ぶね。」

### 先生

- **トーン**: 中立、事務的すぎない
- **スキャン**: F パターン — タイムライン → 不一致 → 不明点
- **色分け**:
  - 事実（facts）— スレート / 青灰
  - 気持ち（feelings）— 温色（オレンジは urgent のみ）
  - 不明（unknowns）— 点線ボーダー、グレー

## 禁止 UI パターン

- `guilty`, `verdict`, `winner`, 悪い子, 有罪
- 子ども A/B の赤緑対比（どちらが正しいか示唆）
- disclaimer の折りたたみ
- セッション ID の子ども画面への露出

## VRM 男女選択（Q2 確定）

- CharaTomo-Web 既存モデル流用
- **男性** / **女性** をセッション開始前に選択
- デフォルト: 前回選択 or 女性（CharaTomo デフォルトと同様）
- 切替時: `reloadVRMModel` 相当 — 同一 URL ならスキップ

## Kebbi との整合

| 項目 | Web | Kebbi |
|------|-----|-------|
| キャラトーン | 黒子・supportive | 同じプロンプト応答 |
| 物理性 | VRM 画面内 | ロボット本体 |
| デモ | 同一 API `/v1/sessions/*` | `NakanaoriApi.kt` |

Web で選んだ男女は **Web 表示専用**（API には送らない MVP）。将来 Phase 2 で preference API 化可。

## アクセシビリティ（MVP 最低限）

- チャット: `role="log"` + 新着 `aria-live="polite"`
- ボタン: ラベル必須（アイコンのみ不可）
- VRM フォールバック: 2D 画像に `alt="話を聞くロボット"`
- 色だけに依存しない urgent 表示（アイコン + 文言）
