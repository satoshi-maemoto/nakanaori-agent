# 緊急エスカレーションストーリー — デモ用例文

**高リスク表現**（暴力・いじめ・自傷など）を含む発話で、ナカナオリが **仲介を止めて先生へ即時エスカレーション** する動作を確認するための台本です。

> ⚠️ **デモ専用** — 実際の児童の被害を演じる場合は配慮してください。  
> ロボットは **裁かない・解決しない**。「先生を呼ぶ」だけです。

> 通常の仲介デモ: [turn-order-story-dialogue.md](./turn-order-story-dialogue.md)（順番取り合い）

---

## このデモで見せること

| 項目 | 期待動作 |
|------|----------|
| 子ども UI | 「だいじょうぶ？ せんせいを よぶね。」→ **入力停止**、「せんせいを よんでね」バナー |
| セッション | `state: escalated`、`escalated: true` |
| 2人目の番 | **進まない**（自律的に仲介続行しない） |
| 先生 UI | **緊急** 表示、`escalation_reason`、`urgent: true` のブリーフ |
| ロボット | 有罪認定・処罰提案 **なし** |

実装: `EmotionGuardAgent` → `packages/agents/src/agents/patterns.ts`

---

## 検出パターン（MVP）

以下を含む発話で **即時エスカレーション**（部分一致）:

| カテゴリ | トリガー例 |
|----------|------------|
| 暴力 | `殴った` `殴って` `殺す` `怪我` `血が` |
| いじめ | `いじめ` |
| 自傷 | `自殺` `死にたい` |

**注意**: `殴られた` だけでは **検出されない** 場合があります。デモでは **`殴った`** または **`殴って`** を含めてください。

## 推奨台本 A — 暴力（殴った）

**2回「おくる」でエスカレーション**。3回目以降は不要（入力不可）。

| # | 操作 | 送る言葉 | 期待 |
|---|------|----------|------|
| 1 | おくる | `さくら` | 名前登録 |
| 2 | おくる | `休み時間、ブランコの 順番で けんか して、たくみくんが わたしの 顔を 殴った。痛かった。` | **即エスカレーション** |
| — | （以降） | 入力不可 | 「せんせいを よんでね」 |

**Kebbi**: 2 回目を音声入力。エスカレーション後は ASR 停止。

### 子ども画面で確認

1. ロボット: 「だいじょうぶ？ せんせいを よぶね。」（TTS）
2. システムメッセージ: 「だいじょうぶ？ せんせいを よぶね。」
3. バナー: 「せんせいを よんでね。 ここで まってて。」
4. 「おくる」「番を おわる」が **使えない**

### 先生画面で確認

1. 進行中セッション一覧 — 状態 **「先生対応が必要」**、**緊急** バッジ
2. セッション詳細 — `escalation_reason`（例: 高リスク表現を検出）
3. 先生ブリーフ — `urgent: true`、`teacher_hints` に早めの確認・安全確認
4. AI 免責 — あり

## ナカナオリがあったから — 緊急時のストーリー

### もし ナカナオリが いなかったら

```text
さくら: 「たくみくん 殴った！」（泣きながら）
たくみ: 「違う！」
（クラス中にエスカレート、先生が気づくまで時間がかかる）
```

### ナカナオリが あったから

```text
1. さくらが ロボットに向かって 「殴った」と 話す
2. ロボット: 「だいじょうぶ？ せんせいを よぶね。」— 仲介続行しない
3. さくら: 「せんせいを よんでね。 ここで まってて。」— 入力停止
4. 先生: ダッシュボードで 緊急 表示 → 即座に児童のもとへ
5. ロボットは 「誰が悪い」とは言わない。先生が安全確認
```

**epilogue（審査員向け）**:

> 「暴力の話は、ロボットが **自分で解決しようとしない**。  
> 即座に先生へ渡す — **子ども安全のための設計** です。」

## curl 確認

```bash
SID=$(curl -sf -X POST http://localhost:8080/v1/sessions \
  -H "Content-Type: application/json" \
  -d '{"child_a_label":"子どもA","child_b_label":"子どもB"}' | jq -r '.session_id')

# 名前
curl -sf -X POST "http://localhost:8080/v1/sessions/$SID/child-turn" \
  -H "Content-Type: application/json" \
  -d '{"child_id":"a","utterance":"さくら"}'

# エスカレーション
curl -sf -X POST "http://localhost:8080/v1/sessions/$SID/child-turn" \
  -H "Content-Type: application/json" \
  -d '{"child_id":"a","utterance":"たくみくんが ぼくの 顔を 殴った。"}' | jq '{escalated, agent_message, state}'

# 期待: escalated=true, agent_message="大丈夫？先生を呼ぶね。", state="escalated"

curl -sf "http://localhost:8080/v1/sessions/$SID/teacher-brief" | jq '{urgent, teacher_hints, ai_disclaimer}'
```

## 実装上の注意（デモ担当向け）

- エスカレーション発話は **EmotionGuard が先に判定** するため、その発話自体は `turns_a` に **残らない** 場合があります（`escalation_reason` にパターン情報）
- 先生ブリーフは `formatEscalationBrief` — 通常の fact/feelings 整理より **短い緊急版**
- 禁止: ロボットが「たくみくんが悪い」等の **裁き** を出力しないこと（プロダクトルール）
